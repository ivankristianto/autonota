# Nota Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Node CLI named `nota` that downloads YouTube audio, chunks oversized MP3s, transcribes with OpenAI Whisper into JSON, and summarizes transcript JSON into Markdown.

**Architecture:** Use a small ESM TypeScript CLI with two command modules and reusable helpers for YouTube download, audio chunking, transcription, summary generation, filesystem handling, and terminal status output. Keep command modules thin, move pipeline logic into focused `src/lib/*` units, and preserve the proven chunking and retry behavior from the existing `transcribe-openai.js` script while removing project-specific assumptions.

**Tech Stack:** Node.js, TypeScript, Commander, Listr2, OpenAI Node SDK, Vitest, tsx

---

## File Structure

Planned files and responsibilities:

- `package.json`: package metadata, scripts, runtime dependencies, `nota` bin entry
- `tsconfig.json`: ESM TypeScript build config for `src/` to `dist/`
- `.gitignore`: ignore build output, temp files, env files, generated artifacts
- `README.md`: usage, prerequisites, examples, and environment setup
- `AGENTS.md`: project guidance for future coding agents
- `.env.example`: documented environment variable names
- `src/cli.ts`: Commander entrypoint and top-level error handling
- `src/types.ts`: transcript types, summary options, and artifact contracts
- `src/commands/transcribe.ts`: orchestration for the transcription pipeline
- `src/commands/summarize.ts`: orchestration for Markdown summary generation
- `src/lib/fs.ts`: output-path derivation, overwrite checks, JSON read/write helpers
- `src/lib/requirements.ts`: dependency and environment validation
- `src/lib/youtube.ts`: YouTube URL normalization, metadata extraction, and audio download
- `src/lib/audio.ts`: duration probing, file-size checks, chunk planning, split cleanup
- `src/lib/transcription.ts`: Whisper calls, rate-limit retries, transcript assembly
- `src/lib/summary.ts`: prompt creation, model call, Markdown formatting
- `src/lib/tui.ts`: lightweight task/spinner helpers for command progress
- `tests/unit/fs.test.ts`: path and overwrite behavior
- `tests/unit/youtube.test.ts`: URL normalization and video-id extraction
- `tests/unit/audio.test.ts`: chunk sizing math and chunk-plan behavior
- `tests/unit/requirements.test.ts`: fail-fast requirement validation behavior
- `tests/unit/transcription.test.ts`: segment normalization and transcript assembly
- `tests/unit/summary.test.ts`: Markdown formatter and prompt contract
- `tests/commands/transcribe.test.ts`: transcribe command with mocked subprocess/OpenAI behavior
- `tests/commands/summarize.test.ts`: summarize command with mocked OpenAI behavior
- `tests/fixtures/sample-transcript.json`: stable transcript input for summary tests
